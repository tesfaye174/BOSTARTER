Premessa e Specifiche del Progetto

BOSTARTER - Piattaforma di Crowdfunding

Obiettivo

Realizzare una piattaforma web per supportare campagne di crowdfunding finalizzate alla realizzazione di progetti hardware/software, ispirata a Kickstarter.

Utenti della Piattaforma

  • Utenti Normali: Registrazione, finanziamento, commenti
  • Creatori: Creazione progetti, gestione candidature
  • Amministratori: Gestione competenze, popolamento dati

Tipi di Progetto

  • Hardware: Lista componenti con prezzo/quantità
  • Software: Profili richiesti con skill specifiche

Meccanismi di Interazione

  • Finanziamento progetti con selezione reward
  • Candidature per progetti software (skill-matching)
  • Sistema commenti e risposte
  • Logging eventi in MongoDB

Glossario dei Dati

Definizioni e significati dei termini utilizzati

Utente

Persona registrata alla piattaforma che può finanziare progetti, inserire commenti, candidarsi a posizioni software.

Creatore

Sottotipo di utente che può creare e gestire progetti. Dispone di metriche come affidabilità e numero progetti.

Amministratore

Sottotipo di utente con privilegi speciali per gestire la lista globale delle competenze.

Progetto

Iniziativa creata da un utente creatore per raccogliere fondi. Può essere hardware o software.

Reward

Ricompensa non economica offerta ai finanziatori in cambio del loro supporto economico.

Finanziamento

Contributo economico fornito da un utente per supportare un progetto, associato a una reward.

Candidatura

Richiesta di partecipazione a un progetto software da parte di un utente che soddisfa i requisiti di skill.

Skill

Competenza tecnica valutata su scala 0-5, utilizzata per matching tra candidati e posizioni richieste.

Business Rules

Regole di business e vincoli del sistema

BR1: Chiusura Progetto per Budget

Un progetto diventa CHIUSO quando la somma totale dei finanziamenti ricevuti ≥ budget richiesto.

BR2: Chiusura Progetto per Data

Un progetto diventa CHIUSO quando la data corrente supera la data limite impostata.

BR3: Calcolo Affidabilità

Affidabilità = (progetti_con_almeno_un_finanziamento / progetti_totali_creati) × 100

BR4: Skill Matching Candidature

Un utente può candidarsi SOLO se, per ogni skill richiesta dal profilo, ha livello ≥ livello richiesto.

BR5: Risposte Commenti

Ogni commento può ricevere al massimo UNA risposta ufficiale dal creatore del progetto.

BR6: Stato Progetto

Un progetto chiuso non accetta ulteriori finanziamenti né candidature.

Vincoli di Integrità
  • Email e nickname univoci a livello globale
  • Nome progetto univoco
  • Codice reward univoco
  • Commento ID univoco autoincrementante
  • Livello skill compreso tra 0 e 5

Relazioni E-R Dettagliate

Cardinalità e associazioni del modello

Relazioni Principali

UTENTE - CREA - PROGETTO
Un utente (creatore) può creare più progetti
Un progetto è creato da esattamente un utente
Cardinalità: 1:N
PROGETTO - HA - REWARD
Un progetto può avere più rewards
Una reward appartiene a un solo progetto
Cardinalità: 1:N
UTENTE - FINANZIA - PROGETTO
Un utente può finanziare più progetti
Un progetto può ricevere finanziamenti da più utenti
Cardinalità: N:M

Relazioni Social

UTENTE - INSERISCE - COMMENTO
Un utente può inserire più commenti
Un commento è inserito da un solo utente
Cardinalità: 1:N
COMMENTO - RIGUARDA - PROGETTO
Un commento riguarda un solo progetto
Un progetto può avere più commenti
Cardinalità: N:1
UTENTE - CANDIDATO - PROFILO
Un utente può candidarsi a più profili
Un profilo può ricevere candidature da più utenti
Cardinalità: N:M

Relazioni di Ereditarietà

UTENTE - ISA - (AMMINISTRATORE | CREATORE)
Relazione di sottotipo disgiunta: un utente può essere normale, amministratore, creatore, o nessuna delle due categorie speciali.
Specializzazione parziale e disgiunta

Analisi delle Ridondanze

Campo nr_progetti nel modello CREATORE

Ridondanza Identificata

Il campo nr_progetti nella tabella CREATORE è ridondante perché può essere calcolato contando i progetti associati al creatore.

Vantaggi della Ridondanza
  • Performance: Query più veloci per conteggi
  • Interattività: wI = 1 (operazione interattiva)
  • Efficienza: Evita JOIN per conteggi frequenti
Svantaggi della Ridondanza
  • Integrità: Rischio inconsistenza dati
  • Manutenzione: Trigger necessari per aggiornamenti
  • Spazio: Dati duplicati

Analisi dei Volumi

Operazione Frequenza Costo senza ridondanza Costo con ridondanza
Inserimento progetto 1/mese 1 (INSERT) 1 (INSERT) + 1 (UPDATE trigger)
Visualizzazione progetti 1/mese N (JOIN) 1 (SELECT diretto)
Conteggio progetti utente 3/mese 1 (COUNT + JOIN) 1 (SELECT diretto)
Decisione: Mantenere la ridondanza nr_progetti per ottimizzare le performance delle operazioni interattive più frequenti.

4. Normalizzazione

Analisi della forma normale del database

Forma Normale 1 (1NF)

Requisiti soddisfatti:
✓ Valori atomici in ogni colonna
✓ Nessun gruppo ripetuto
✓ Chiave primaria identificata
✓ Relazioni tra tabelle stabilite

Forma Normale 2 (2NF)

Requisiti soddisfatti:
✓ In 1NF
✓ Nessuna dipendenza parziale da chiave
✓ Tutti gli attributi non-chiave dipendono completamente dalla chiave primaria

Forma Normale 3 (3NF)

Requisiti soddisfatti:
✓ In 2NF
✓ Nessuna dipendenza transitiva
✓ Eliminazione ridondanze
✓ Normalizzazione BCNF raggiunta

Considerazioni

Ridondanza controllata:
- Campo nr_progetti mantenuto per performance
- Trigger automatici per consistenza
- Viste per statistiche ottimizzate

Trade-off Normalizzazione vs Performance

Schema normalizzato per integrità, con compromessi calcolati per l'efficienza operativa del sistema di crowdfunding.

Appendice B: Tabelle Progetto

Progetti, rewards e componenti

2. Tabelle Progetto
-- Tabella Progetti
CREATE TABLE progetti (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nome VARCHAR(255) UNIQUE NOT NULL,
    descrizione TEXT,
    creatore_id INT NOT NULL,
    tipo ENUM('hardware', 'software') NOT NULL,
    budget_richiesto DECIMAL(10,2) NOT NULL,
    totale_raccolto DECIMAL(10,2) DEFAULT 0.00,
    data_inserimento DATETIME DEFAULT CURRENT_TIMESTAMP,
    data_limite DATE NOT NULL,
    stato ENUM('aperto', 'chiuso') DEFAULT 'aperto',
    FOREIGN KEY (creatore_id) REFERENCES utenti(id) ON DELETE CASCADE
);

-- Tabella Rewards
CREATE TABLE rewards (

API Candidature - Backend PHP
<?php
/**
 * API per la gestione delle candidature ai progetti software
 */
session_start();
require_once __DIR__ . '/../autoload.php';

header('Content-Type: application/json');

$roleManager = new RoleManager();
$apiResponse = new ApiResponse();

// Verifica autenticazione
if (!$roleManager->isAuthenticated()) {
    $apiResponse->sendError('Autenticazione richiesta', 401);
    exit();
}

switch ($_SERVER['REQUEST_METHOD']) {
    case 'GET':
        // Lista candidature per profilo o utente
        if (isset($_GET['profilo_id'])) {
            $profiloId = (int)$_GET['profilo_id'];
            $candidature = getCandidaturePerProfilo($profiloId, $roleManager);
            $apiResponse->sendSuccess($candidature);
        } elseif (isset($_GET['utente_id'])) {
            $utenteId = (int)$_GET['utente_id'];
            if ($roleManager->getUserId() != $utenteId && !$roleManager->isAdmin()) {
                $apiResponse->sendError('Accesso negato', 403);
                exit();
            }
            $candidature = getCandidatureUtente($utenteId);
            $apiResponse->sendSuccess($candidature);
        } else {
            // Tutte le candidature dell'utente corrente
            $candidature = getCandidatureUtente($roleManager->getUserId());
            $apiResponse->sendSuccess($candidature);
        }
        break;

    case 'POST':
        // Nuova candidatura
        $input = json_decode(file_get_contents('php://input'), true);

        if (!$input) {
            $apiResponse->sendError('Dati JSON non validi');
            exit();
        }

        $required = ['profilo_id', 'motivazione'];
        foreach ($required as $field) {
            if (!isset($input[$field]) || empty($input[$field])) {
                $apiResponse->sendError("Campo obbligatorio mancante: $field");
                exit();
            }
        }

        $result = creaCandidatura($input, $roleManager->getUserId());
        if ($result['success']) {
            $apiResponse->sendSuccess($result['data'], 'Candidatura inviata con successo');
        } else {
            $apiResponse->sendError($result['error']);
        }
        break;

    case 'PUT':
        // Aggiorna stato candidatura (solo creatore/admin)
        $input = json_decode(file_get_contents('php://input'), true);

        if (!isset($input['candidatura_id']) || !isset($input['stato'])) {
            $apiResponse->sendError('ID candidatura e stato richiesti');
            exit();
        }

        $result = aggiornaStatoCandidatura($input['candidatura_id'], $input['stato'], $roleManager);
        if ($result['success']) {
            $apiResponse->sendSuccess($result['data'], 'Stato candidatura aggiornato');
        } else {
            $apiResponse->sendError($result['error']);
        }
        break;

    case 'DELETE':
        // Cancella candidatura (solo proprietario o admin)
        $candidaturaId = (int)$_GET['id'];
        $result = cancellaCandidatura($candidaturaId, $roleManager);
        if ($result['success']) {
            $apiResponse->sendSuccess(null, 'Candidatura cancellata');
        } else {
            $apiResponse->sendError($result['error']);
        }
        break;

    default:
        $apiResponse->sendError('Metodo non supportato', 405);
        break;
}

// Gestione form candidatura
document.getElementById('candidaturaForm')?.addEventListener('submit', function(e) {
    e.preventDefault();

    const formData = new FormData(this);
    const errors = validateCandidaturaForm(formData);

    if (errors.length) {
        showMessage('error', 'Errore validazione: ' + errors.join(', '));
        return;
    }

    const data = {
        profilo_id: formData.get('profilo_id'),
        motivazione: formData.get('motivazione')
    };

    fetch('/BOSTARTER/backend/api/candidature.php', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'X-CSRF-TOKEN': getCSRFToken()
            },
            body: JSON.stringify(data)
        })
        .then(response => {
            if (!response.ok) {
                throw new Error(`HTTP ${response.status}`);
            }
            return response.json();
        })
        .then(data => {
            if (data.success) {
                showMessage('success', data.message || 'Candidatura inviata!');
                this.reset();
                document.getElementById('profilo_id').disabled = true;
                document.getElementById('profilo_id').innerHTML = '<option value="">Prima seleziona progetto...</option>';
                setTimeout(() => window.location.reload(), 1500);
            } else {
                showMessage('error', data.error || 'Errore invio candidatura');
            }
        })
        .catch(error => handleApiError(error, 'invio candidatura'));
});

// Carica profili progetto
document.getElementById('progetto_id')?.addEventListener('change', function() {
    const progettoId = this.value;
    const profiloSelect = document.getElementById('profilo_id');

    if (!progettoId) {
        profiloSelect.disabled = true;
        profiloSelect.innerHTML = '<option value="">Prima seleziona progetto...</option>';
        return;
    }

    profiloSelect.disabled = true;
    profiloSelect.innerHTML = '<option value="">Caricamento profili...</option>';

    fetch(`/BOSTARTER/backend/api/project.php?id=${progettoId}`)
        .then(response => {
            if (!response.ok) {
                throw new Error(`HTTP ${response.status}`);
            }
            return response.json();
        })
        .then(data => {
            if (data.success) {
                const progetto = data.data;
                if (progetto.profili_richiesti && progetto.profili_richiesti.length > 0) {
                    profiloSelect.innerHTML = '<option value="">Seleziona profilo...</option>';
                    progetto.profili_richiesti.forEach(profilo => {
                        const option = document.createElement('option');
                        option.value = profilo.id;
                        option.textContent = profilo.nome;
                        profiloSelect.appendChild(option);
                    });
                    profiloSelect.disabled = false;
                } else {
                    profiloSelect.innerHTML = '<option value="">Nessun profilo disponibile</option>';
                    profiloSelect.disabled = true;
                }
            } else {
                throw new Error(data.error || 'Errore caricamento profili');
            }
        })
        .catch(error => {
            profiloSelect.innerHTML = '<option value="">Errore caricamento</option>';
            profiloSelect.disabled = true;
            handleApiError(error, 'caricamento profili');
        });
});
